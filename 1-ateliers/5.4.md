# Atelier 5.4 : authentification

## Enoncé

1. Créez un nouveau projet *Express* avec le *CLI* et utilisez le standard *CommonJS*.
2. Traitez l'inscription d'un nouvel utilisateur à partir d'une requête du client avec la méthode **POST** et des données envoyées au format JSON.
- Les nouvelles inscriptions sont stockées dans un fichier JSON ***users.json*** à créer et positionner librement dans votre projet
- Les informations **email et mot de passe sont obligatoires**
- Les autres informations tels que le *nom, prénom, âge, etc.* sont facultatives
- Le mot de passe stocké dans le fichier ***users.json*** est **haché**
- Exemple des données envoyées : route  `/users/sign-up` avec la méthode `POST` et les données au format JSON 
```json
{ 
  "email": "john@doe.com",
  "password": "admin",
  "name": "john",
  "age": 45
}
```
- Lorsque tout s'est bien déroulé, votre serveur répond au format JSON avec le code ***HTTP 201***
```json
{
  "message":"success"
}
``` 

- Dans le cas contraire, votre serveur répond avec le code ***HTTP 400*** et le message associé à l'erreur, par exemple, 
```json
{
  "message":"password required"
}
```
2. Traitez la connexion d'un utilisateur depuis la route `/users/sign-in` en ***POST*** avec son **email** et **mot de passe** envoyées par le client au format en JSON.
- Si le mot de passe envoyé en clair et le mot de passe stocké dans le fichier JSON ***users.json*** pour cet utilisateur correspondent, le serveur répond avec le code HTTP ***200*** et le message JSON avec le  ***token*** généré à partir de ***jsonwebtoken*** dans votre réponse.

```json
{
  "message":"success",
  "token":"generate_token_here"
}
```
